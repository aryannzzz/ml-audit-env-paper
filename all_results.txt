7. STRUCTURAL OUTLIERS / FAILURE MODES IDENTIFIED IN LOGS
================================================================================
(a) Cross-tier RNG correlation (pre-fix v1.0):
    random.seed(seed) without per-tier offset caused identical episode
    indices across easy/medium/hard. Seed 43 drew timeseries_reg_104
    (clean) for all three tiers simultaneously. Fix: per-tier offsets
    (easy=0, medium=10000, hard=20000). Empirical effect on GPT-4.1-mini
    medium variance: pre-fix sigma=0.553 -> post-fix (seeds 42-44) sigma=0.009.

(b) V3 false-positive trap on clean timeseries_reg_104:
    feature-engineering snippet referenced X_train['energy_demand_kwh']
    (the target), causing every model to correctly raise V3 on a
    nominally clean episode. Replaced with weather-lag features.

(c) Duplicate flag penalty:
    Models occasionally re-flagged the same violation type after a
    partial reward. Each duplicate cost -0.10. Mitigation: tracked
    flagged_violation_types set in inference.py.

(d) Evidence-quote fabrication (reasoning models):
    o4-mini at seed 42 (older inference.py) submitted correct verdict
    with paraphrased quote -> failed three-layer matcher, scored 0.10.
    Mitigation: 10-60 char verbatim guidance + 6 surfaced excerpts
    + flagless-reject downgrade. Post-fix: 0.912 at seed 42.

(e) Adversarial clean-episode concentration on easy tier:
    Seeds 44, 47, 49, 50, 51 all draw clean episodes on easy -> every
    model scores 0.160 (verdict-bonus only, since they submitted reject).
    Five out of ten seeds at the headline-tier is the largest single
    source of per-seed variance -- and is by design.

(f) o4-mini self-consistent hallucination (seed 48):
    Easy 0.100, medium 0.008, hard 0.039 -- the chain-of-thought
    iteratively reinforces an incorrect initial hypothesis (e.g.
    misreading GroupShuffleSplit as entity leakage) and produces
    fabricated quotes consistent with that hypothesis. Each
    fabrication costs -0.05; the episode collapses.

10. VARIANCE REDUCTION STUDY  (inference.py v2 — 2026-04-28/29)
    Model: GPT-4.1-mini | Seeds 42-51 | Sequential (temperature=0)
================================================================================

Goal: reduce per-seed score variance on easy and medium tiers to near-zero.
Root-cause analysis found five additional inference-script failure modes beyond
the four documented in Section 7 (a)-(d) above.

--------------------------------------------------------------------------------
10.1  Failure modes diagnosed and fixes applied
--------------------------------------------------------------------------------

(e) ENFORCE_COMPARE exhausting last budget slot  [medium seeds 48, 51 — V3+V6]
    The forced compare logic (ENFORCE_COMPARE=1) could fire at step_index=11 on
    12-step medium episodes, leaving the post-loop submit as step 13. env.py
    auto-terminates at steps_used > step_budget with verdict="reject", scoring
    the episode as a false-reject on a real V3+V6 experiment.
    FIX A: _required_compare_action now returns None when step_index >= budget-2
           ("Reserve last 2 steps for flag + submit; don't consume with forced compares").
    FIX B: Last-step guard extended to block both "inspect" AND "compare" actions
           (previously only blocked "inspect").

(f) Verdict computed from model intent, not actual flags  [medium seed 42 — V1+V5]
    Agent correctly flagged V1 (reject severity) but submitted verdict="revise"
    (driven by V5 reasoning), forfeiting the full verdict bonus.
    FIX: _normalize_action now overrides submitted verdict from flags_raised in
         observation, using _smart_verdict() to compute canonical verdict from
         the actual flagged violation types.

(g) Premature submit from "steps_remaining ≤ 1" rule  [medium seed 42]
    Agent submitted at steps_remaining=1, forfeiting the opportunity to flag V5
    on the next step.
    FIX: Submit timing changed from "steps_remaining ≤ 1" to "steps_remaining == 0"
         throughout SYSTEM_PROMPT and RULES.

(h) Deferred-flagging strategy caused missed V1  [medium seed 47 — V1+V8]
    Old INSPECTION STRATEGY said "Step 3 (late budget): Flag" — agent saw clear
    V1 evidence in preprocessing at step 4 but deferred all flags to late budget,
    then ran out of steps before flagging V1.
    FIX: INSPECTION STRATEGY completely rewritten:
         "FLAG IMMEDIATELY when you find clear evidence — then keep inspecting
          remaining artifacts." Flag immediately, don't defer.

(i) V5 disclosure false-negative  [medium seed 42]
    Experiment notes contained "Final model selected with fixed seed". Agent
    treated the word "selected" as proper disclosure even though neither the
    run count nor the selection metric were stated.
    FIX: SYSTEM_PROMPT now explicitly defines proper V5 disclosure as requiring
         BOTH the exact run count AND the selection metric, with a named negative
         example: '"Final model selected with fixed seed" → NOT proper disclosure
         (no run count, no metric) → V5'.

Additional tuning (all medium tier):
  - Reminder threshold: steps_remaining >= 2 → >= 3, wording changed to
    "Keep inspecting — there may be more violations" (was "inspect remaining
    artifacts first"). Prevents reminder from suppressing V6 flagging after
    ENFORCE_COMPARE compare result on V3+V6 seeds.
  - Neutral goal field: removed "Inspect ALL artifacts" opening that conflicted
    with new flag-immediately strategy.
  - Inspection priority order in SYSTEM_PROMPT aligned with ACTION_PRIORITY
    code: validation_strategy/eval_report moved before run_history/experiment_notes.
